$ mkdir /home/bagira/test
$ touch /home/bagira/test/test-file
$ rm -r /home/bagira/test

Как и в случае с командой rm, вы можете задать параметр -r для команд cp и mv. В этом случае операция копирования или перемещения будет выполняться рекурсивно.

Очень важной операцией является просмотр содержимого каталога, для которой используется команда ls. Поэтому сейчас сделаем исключением для этой команды и рассмотрим ее параметры (табл. 4.3). А общий формат вызова этой команды таков:

ls [параметры] [каталог]

Таблица 4.3. Параметры команды ls

<table>
  <tr>
    <th>Параметр</th>
    <th>Описание</th>
  </tr>
  <tr>
    <td>-C</td>
    <td>Выводит список файлов в колонках с вертикальной сортировкой</td>
  </tr>
  <tr>
    <td>-F</td>
    <td>Для каждого каталога добавлять суффикс '/', а для каждого исполняемого файла - '*', а для каждого FIFO-канала - '|'</td>
  </tr>
  <tr>
    <td>-R</td>
    <td>Рекурсивный вывод, то есть команда ls будет выводить не только содержимое каталога, но и подкаталогов</td>
  </tr>
  <tr>
    <td>-a</td>
    <td>Показывать скрытые файлы.</td>
  </tr>
  <tr>
    <td>-i</td>
    <td>Показывать иноды для каждого файла (будет показан серийный номер файла)</td>
  </tr>
  <tr>
    <td>-l</td>
    <td>«Длинный» формат вывода, в котором отображается тип файла, права доступа, количество ссылок на файл, имя владельца, имя группы, размер файла, метка времени создания файла и имя файла.<br>В колонке типа файла могут быть следующие значения: d (каталог), b (блочное устройство), c (символьное устройство), l (символическая ссылка), p (FIFO-канал), s (сокет).</td>
  </tr>
  <tr>
    <td>-r</td>
    <td>Сортировка в обратном порядке</td>
