англ. change attribute) и вывести список этих атрибутов с помощью команды lsattr.

Как и в случае с командой chmod, атрибуты можно устанавливать (+) или добавлять (-) относительно, или устанавливать абсолютно (=).

<table>
  <tr>
    <th>Атрибут</th>
    <th>Значение</th>
  </tr>
  <tr>
    <td>a</td>
    <td>Файл только для добавления данных, т. е. разрешается только добавлять данные и нельзя редактировать как-либо еще. Только для суперпользователя</td>
  </tr>
  <tr>
    <td>A</td>
    <td>Доступ к файлу не фиксируется во временных метках: доступ к файлу не обновляет его временную метку доступа (atime)</td>
  </tr>
  <tr>
    <td>c</td>
    <td>Сжатый: данные сжимаются при записи и распаковываются при чтении</td>
  </tr>
  <tr>
    <td>d</td>
    <td>Неархивируемый: программа dump должна игнорировать этот файл при резервном копировании (см. раздел "Резервное копирование и удаленное хранение" на странице 154)</td>
  </tr>
  <tr>
    <td>i</td>
    <td>Неизменяемый: файл не может быть изменен или удален (только для суперпользователя)</td>
  </tr>
  <tr>
    <td>j</td>
    <td>Журналируемые данные (только для файловой системы ejh)</td>
  </tr>
  <tr>
    <td>s</td>
    <td>Безопасное удаление: в случае удаления данные этого файла переписываются нулями</td>
  </tr>
  <tr>
    <td>s</td>
    <td>Синхронное обновление: изменения записываются на диск немедленно, как будто вы набираете команду sync после сохранения (см. раздел "Диски и файловые системы" на странице 146)</td>
  </tr>
  <tr>
    <td>u</td>
    <td>Неудаляемый: файл не может быть удален</td>
