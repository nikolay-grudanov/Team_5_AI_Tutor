      <b>--version</b><br>
      Вывести номер версии программы на стандартный вывод и завершить работу.<br>
      <b>-w, --no-character-count</b><br>
      После распаковки не проверять каждую строку командой wc -c. По умолчанию проверка выполняется.<br>
      <b>-x, --no-check-existing</b><br>
      Перезаписывать существующие файлы. По умолчанию производится проверка, и существующие файлы сохраняются. Если -c передается в качестве параметра сценария при распаковке (sh archive -c), происходит безусловная перезапись файлов. См. также -X.<br>
      <b>-z, --gzip</b><br>
      Обработать файлы gzip и uuencode, прежде чем поместить их в архив. Распаковка должна осуществляться с помощью uudecode и gunzip (или zcat).<br>
      <b>-B, --uuencode</b><br>
      Считать все файлы двоичными; выполнить uuencode перед упаковкой файлов. Это увеличивает размер архива. Для извлечения файлов необходимо запустить uudecode.<br>
      <b>-D, --no-md5-digest</b><br>
      Не использовать подпись «md5sum» для проверки распакованных файлов. По умолчанию проверка производится.<br>
      <b>-F, --force-prefix</b><br>
      Принудительное добавление префиксного символа к каждой строке, даже если это не требуется. Может незначительно увеличить размер архива, в особенности при использовании совместно с параметром -B или -Z.<br>
      <b>-L nn, --split-size-limit=nn</b><br>
      Ограничить размер архива nn килобайтами и разделять файлы по необходимости. Части архива, созданные в этом режиме, должны распаковываться в корректном порядке. Требуется использование параметра -o.<br>
      <b>-M, --mixed-uuencode</b><br>
      Упаковка файлов в смешанном режиме (по умолчанию). Различаются текстовые и двоичные файлы; двоичные файлы кодируются uuencode перед помещением в архив.<br>
      <b>-P, --no-piping</b><br>
      Использовать временные файлы вместо конвейеров.
