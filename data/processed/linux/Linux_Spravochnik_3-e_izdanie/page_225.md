    <th>ipchains</th>
    <td>
      <b>-n, --numeric</b><br>
      Отображать все IP-адреса и номера портов в численном виде. По умолчанию, где возможно, отображаются имена.<br>
      <b>-l, --log</b><br>
      При соответствии пакета правилу создавать соответствующую запись в системном журнале.<br>
      <b>-t andmask xormask, --TOS andmask xormask</b><br>
      Изменить поле Type of Service (Тип службы) в заголовке пакета. Поле пакета TOS — первое, над которым выполняется операция логического умножения AND на восьмибитную маску <i>andmask</i>, затем результат используется в качестве первого операнда для XOR <i>xormask</i>, где <i>xormask</i> — также восьмибитная маска. Правила, которые могут повлиять на наименее значимый бит (LSB) поля TOS, не применяются.<br>
      <b>-x, --exact</b><br>
      Производить подстановку чисел при перечислении (<b>-L</b>). Отображать точные значения счетчиков байт и пакетов вместо округленных.<br>
      [!] <b>-y, --syn</b><br>
      Отбор только входящих запросов на TCP-соединение, т. е. запросов с установленным битом SYN и сброшенными битами ACK и FIN. Таким образом, входящие TCP-соединения блокируются, но не затрагиваются исходящие соединения.<br>
      <b>--line-numbers</b><br>
      Используется совместно с командой <b>-L</b>. Добавлять номер строки в начало каждого правила в перечислении, определяя, таким образом, позицию правила в цепи.<br>
      <b>--no-warnings</b><br>
      Отключить выдачу предупреждений.
    </td>

Восстановить набор правил брандмауэра. ipchains-restore использует для восстановления правил брандмауэра в каждой цепи команды, сгенерированные ipchains-save. Часто применяется в сценариях инициализации для восстановления набора правил при загрузке системы.
