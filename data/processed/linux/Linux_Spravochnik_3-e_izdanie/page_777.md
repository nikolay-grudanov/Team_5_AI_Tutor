Вывести журнал событий для всех файлов.

Параметр -l имеет тот же смысл, что и обычно. Дополнительные параметры перечислены в табл. 14.30.

Таблица 14.30. Параметры log

<table>
  <tr>
    <th>Параметр</th>
    <th>Описание</th>
  </tr>
  <tr>
    <td>-b</td>
    <td>Перечислять версии ветви по умолчанию</td>
  </tr>
  <tr>
    <td>-d dates</td>
    <td>Сообщить информацию для указанных дат</td>
  </tr>
  <tr>
    <td>-h</td>
    <td>Вывести только заголовки</td>
  </tr>
  <tr>
    <td>-N</td>
    <td>Не отображать метки</td>
  </tr>
  <tr>
    <td>-r[revisions]</td>
    <td>Отчет только по перечисленным версиям. Пробела между -r и аргументом нет. Параметр без аргумента является указанием использовать последнюю версию ветви по умолчанию</td>
  </tr>
  <tr>
    <td>-R</td>
    <td>Выводить только имя файла RCS. Использование -R в данном случае имеет иной смысл, чем во всех остальных командах CVS, где параметр -R является предписанием рекурсивного обхода</td>
  </tr>
  <tr>
    <td>-s state</td>
    <td>Выводить только версии, находящиеся в указанном состоянии</td>
  </tr>
  <tr>
    <td>-t</td>
    <td>Выводить только заголовки и текст описания</td>
  </tr>
  <tr>
    <td>-wlogins</td>
    <td>Отчет по помещению в репозиторий файлов перечисленными пользователями. Пробела между -w и аргументом нет</td>
  </tr>
</table>

С параметром -d следует использовать спецификации дат, приведенные в табл. 14.31. Допустимо указание нескольких спецификаций, разделенных точкой с запятой.
