      <i>site.def</i><br>
      Содержит специфичные для проекта данные, такие как каталоги установки, набор компилируемых программ, использование специальных версий программ при сборке проекта. На разных машинах файл <i>site.def</i> различается.<br>
      <i>Project.tmpl</i><br>
      Файл, содержащий переменные, специфичные для X.<br>
      <i>Library.tmpl</i><br>
      Файл, содержащий правила работы с библиотеками.<br>
      <i>Server.tmpl</i><br>
      Файл, содержащий правила, специфичные для сервера.<br>
      <i>.cf</i> Файлы <i>.cf</i> (<i>VendorFiles</i>, расположенные в <i>Imake.vb</i>) содержат настройки разработчика. В каждом таком файле находится информация о платформе разработки, в частности, о версии операционной системы и компилятора, а также альтернативные пути реализации отсутствующих возможностей. Определения в файлах <i>.cf</i> имеют более высокий приоритет, чем соответствующие значения по умолчанию, описанные в файле <i>Imake.params</i>.

      <b>Файл Imakefile</b><br>
      Файл <i>Imakefile</i>, существующий в каждом из каталогов проекта, определяет цели сборки, установки и правила, которым необходимо следовать. Команда imake читает <i>Imakefile</i> и создает из правил целевые записи для файла <i>Makefile</i>. Каждый <i>Imakefile</i> также может содержать определение переменных для make и список зависимостей целей. Зависимости выражаются в виде макросов cpp, определенных в файле <i>Imake.rules</i>. При изменении какого-либо файла <i>Imakefile</i> необходимо произвести пересборку файла <i>Makefile</i> и зависимостей заголовочных файлов. Более подробно о работе с imake можно прочесть в книге Пола Дюбуа (Paul DuBois) из серии Nutshell Handbook: «Software Portability with imake» (Переносимость программ и imake).
