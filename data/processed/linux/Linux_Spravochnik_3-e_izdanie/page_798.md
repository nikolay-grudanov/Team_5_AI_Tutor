Параметр

-q Не выводить предупреждение, если требуемая информация по ключевым словам не найдена.

Примеры

Если файл prog.c скомпилирован и содержит следующую строку кода:

    char rcsID[] = "$Author: george$"

выводится следующий результат:

    % ident prog.c prog.o
    prog.c:
        $Author: george $
    prog.o:
        $Author: george $

Отобразить ключевые слова для всех файлов RCS (предупреждения не выводить):

    co -p RCS/*,v | ident -q

rcs [options] <files

Административная команда для установки или замены атрибутов по умолчанию файлов RCS. Среди прочего rcs позволяет устанавливать жесткую блокировку (-L), удалять версии (-o), а также обходить блокировку, установленную со (параметры -l и -u). Для файлов RCS существует список доступа (создаваемый указанием параметра -a), в котором перечислены пользователи, имеющие право запускать rcs. Зачастую список доступа пуст; это означает, что rcs могут использовать все. Кроме того, вы всегда можете использовать rcs, если являетесь владельцем файла, привилегированным пользователем или выполняете rcs с параметром -i. rcs работает со стандартными параметрами -q, -V, -x и -z.

Параметры

-a users
    Добавить пользователей users (перечисляются через запятую) в список доступа.

-A otherfile
    Добавить список доступа otherfile к списку доступа для файлов files.
