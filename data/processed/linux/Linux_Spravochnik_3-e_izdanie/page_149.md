    <th>find</th>
    <td>
      <b>-print</b><br>
      Отображать абсолютные пути для всех найденных файлов и каталогов. Код возврата — true.<br>
      <b>-regex pattern</b><br>
      Сходно с <b>-path</b>, но с использованием регулярных выражений в стиле grep вместо шаблонов командного интерпретатора, таких как в <b>-name</b> и <b>-path</b>.<br>
      <b>-size n[c]</b><br>
      Найти файлы, состоящие из n блоков либо, если задан модификатор c, то из n символов.<br>
      <b>-type c</b><br>
      Найти файлы типа c. Возможные значения c: b (специальный файл блочного устройства), c (специальный файл символьного устройства), d (каталог), p (конвейер или именованный канал), l (символическая ссылка), s (сокет) или f (обычный файл).<br>
      <b>-user user</b><br>
      Найти файлы, принадлежащие пользователю user (имя или идентификатор).<br>
      <b>-daystart</b><br>
      Отсчитывать время от начала текущего дня.<br>
      <b>-maxdepth num</b><br>
      Не спускаться по дереву каталогов более чем на num уровней.<br>
      <b>-mindepth num</b><br>
      Производить обработку только на уровнях, расположенных ниже уровня num.<br>
      <b>-noleaf</b><br>
      Обычно find полагает, что для каждого каталога существуют две жесткие ссылки, которые следует игнорировать (это ссылка на имя каталога и на «.»), т. е. реальных каталогов на два меньше, чем жестких ссылок. Критерий <b>-noleaf</b> заставляет find обрабатывать все жесткие ссылки без исключения, что бывает полезно на файловых системах не-Unix систем (жесткая ссылка может на самом деле быть каталогом, который необходимо обрабатывать). В системах Unix этот параметр только замедляет работу программы.
