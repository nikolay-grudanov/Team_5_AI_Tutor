    <td>
      <b>-B, --auto-deconfigure</b><br>
      При удалении пакета автоматически «расконфигурировать»<sup>1</sup> любой другой пакет, который зависит от данного.<br>
      <b>-Doctal, --debug=octal</b><br>
      Включить отладку. Восьмеричное значение octal определяет требуемый уровень подробности отладочной информации. Для получения возможных значений используйте -Dh или --debug=help. Значения могут комбинироваться логическим ИЛИ (OR) для получения желаемого результата.<br>
      <b>-E, --skip-same-version</b><br>
      Не устанавливать пакет, если программа этой версии уже существует в системе.<br>
      <b>--force-list, --no-force-list, --refuse-list</b><br>
      Принудительное выполнение (или запрет на выполнение) операции. Параметры в списке list перечисляются через запятую. Команда --force отображает предупреждение, но работа продолжается. --refuse и --no-force приводят к остановке работы с ошибкой. Параметры для force/refuse следующие:<br>
      architecture<br>
      Выполнять обработку, даже если пакет предназначен для другой архитектуры.<br>
      auto-select<br>
      Выбирать пакеты для установки и отменять выбор для удаления пакетов. По умолчанию включен.<br>
      bad-path<br>
      Некоторые программы отсутствуют в стандартных путях поиска.<br>
      configure-any<br>
      Настройка любого ненастроенного пакета, от которого зависит данный пакет.<br>
      conflicts<br>
      Разрешить установку конфликтующих пакетов. Могут возникать проблемы из-за перезаписи файлов.
