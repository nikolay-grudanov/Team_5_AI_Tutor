    <th>rcs</th>
    <td>
      <b>-b[R]</b><br>
      Установить ветвь версий по умолчанию в R или, если аргумент опущен, в самую высокую ветвь издания.<br>
      <b>-c's'</b><br>
      Использовать строку s в качестве символа комментария для ключевого слова $Log. По умолчанию со расширяет найденные ключевые слова $Log в комментарии, предваряемые символом #. Вы можете, например, установить s в значение .\" для файлов troff или установить s в значение * для программ на С. (Охватывающие последовательности /* и */ придется вставлять вручную до и после $Log.)<br>
      <b>-e[users]</b><br>
      Удалить всех пользователей (или только указанного users) из списка доступа.<br>
      <b>-i</b> Создать (инициализировать) файл RCS, но не помещать в него первую версию.<br>
      <b>-I</b> Диалоговый режим. Запрашивать ввод пользователя, даже если стандартным вводом не является терминал. (Параметр появился в версии RCS 5.)<br>
      <b>-kc</b><br>
      Использовать стиль с в качестве стиля по умолчанию для подстановки ключевых слов. (См. возможные значения с в описании команды со.) Указание параметра -kkv восстанавливает исходный стиль подстановок; все прочие стили не являются совместимыми с версией RCS 4 или более ранними.<br>
      <b>-l[R]</b><br>
      Заблокировать версию R или последнюю версию. -l создает «ретро-блокировку», что полезно, если пользователь по ошибке произвел неверное извлечение версии командой со вместо со -l.<br>
      <b>-L</b> Включить жесткую блокировку (режим по умолчанию). Это означает, что все пользователи, включая и владельца RCS-файла, должны использовать со -l для редактирования файла. Жесткая блокировка рекомендуется при совместном редактировании файлов (см. -U).<br>
      <b>-mR:msg</b><br>
      Использовать строку msg для замены записи в журнале, относящейся к версии R. (Параметр появился в RCS версии 5.6.)
