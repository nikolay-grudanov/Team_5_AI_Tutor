Если версия, указанная для параметра <b>-l</b>, является ветвью, используется самая последняя версия из этой ветви. Если версия не указана, используется последняя версия ветви по умолчанию.

Если имя, указанное для параметра <b>-n</b>, уже используется, выдается сообщение об ошибке. Для перемещения метки можно воспользоваться параметром <b>-N</b> (изменится номер версии, связанной с меткой); однако рекомендуется использовать для этих целей команды <b>cvs tag</b> и <b>cvs rtag</b>.

Использование параметра <b>-o</b> крайне опасно, поскольку приводит к полному удалению информации из репозитория. При его использовании следует проявлять особую осторожность. В табл. 14.10 описаны различные способы задания диапазонов версий. Удаляемая версия не должна иметь ветвлений и не должна быть заблокирована. Также следует иметь в виду возможные взаимодействия этой команды с метками.

Если имя файла <i>file</i> не указано для параметра <b>-t</b>, CVS производит чтение со стандартного ввода, пока не будет достигнут конец файла или прочитан символ точки на отдельной строке.

Для параметра <b>-u</b> версия, над которой производится операция, определяется точно так же, как для <b>-l</b>.
