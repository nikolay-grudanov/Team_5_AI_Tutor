    <th>ipfwadm</th>
    <td>
      <b>-s tcp tcpfin udp</b><br>
      Установить интервалы ожидания для сокрытия адресов.<br>
      Применение корректно только с категорией -M. Все три аргумента являются обязательными и представляют собой значения интервалов ожидания в секундах для TCP-сеансов, TCP-сеансов после получения пакета FIN и UDP-сеансов. Если указан 0, то текущее значение аргумента сохраняется.<br>
      <b>-z</b> Обнулить счетчики байт и пакетов для всех правил в данной категории. Команда может использоваться совместно с командой <b>-l</b>.
    </td>

  <tr>
    <th colspan="2">Настройки</th>
  </tr>
  <tr>
    <td colspan="2">
      С командами <b>-a</b>, <b>-i</b>, <b>-d</b> и <b>-c</b> могут быть указаны следующие параметры. Обратное отмечается специально. Несколько параметров можно задать в одной командной строке ipfwadm.<br>
      <b>-D address[/mask][port...]</b><br>
      Спецификация пункта назначения (необязательная). Синтаксис, значения по умолчанию и прочие требования приведены в описании команды <b>-S</b>. Типы ICMP не могут указываться с <b>-D</b>.<br>
      <b>-P protocol</b><br>
      Протокол правила или пакета; корректные значения: tcp, udp, icmp или all. По умолчанию — all, т. е. все протоколы. <b>-P</b> не может использоваться совместно с командой <b>-c</b>.<br>
      <b>-S address[/mask][port...]</b><br>
      Исходный IP-адрес, указанный в виде имени узла, имени сети или IP-адреса. Исходный адрес и маска по умолчанию имеют вид 0.0.0.0/0. Если указан параметр <b>-S</b>, также должен быть указан и параметр <b>-P</b>. Необязательная маска задается как маска сети либо как число единиц в левой части двоичного представления маски (например, маска 24 эквивалентна 255.255.255.0). По умолчанию значение маски равно 32. Может быть определено одно или несколько необязательных значений port, указывающих, к каким типам ICMP-портов должно применяться правило. По умолчанию port имеет значение all. Порты могут быть указаны в записях файла /etc/services. Синтаксис для указания диапазона портов следующий:<br>
      <i>lowport:highport</i><br>
      Например:<br>
      <b>-S 172.29.16.1/24 ftp:ftp-data</b>
