Настройка CVS

На поведение CVS, помимо аргументов командной строки, могут влиять два класса настроек: переменные окружения (табл. 14.13) и специальные файлы (табл. 14.14).

Таблица 14.13. Переменные окружения

<table>
  <tr>
    <th>Переменная</th>
    <th>Описание</th>
  </tr>
  <tr>
    <td>$COMSPEC</td>
    <td>Командный интерпретатор на системах OS/2, если это не cmd.exe.</td>
  </tr>
  <tr>
    <td>$CVS_CLIENT_LOG</td>
    <td>Отладочный файл на стороне клиента для соединений типа клиент-сервер.<br>$CVS_CLIENT_LOG — основа имени файла для файлов $CVS_CLIENT_LOG.in и $CVS_CLIENT_LOG.out, которые создаются в текущем рабочем каталоге во время выполнения команды.</td>
  </tr>
  <tr>
    <td>$CVS_CLIENT_PORT</td>
    <td>Номер порта для путей формата :kserver:.<br>$CVS_CLIENT_PORT можно не устанавливать, если kserver принимает запросы через порт 1999 (порт по умолчанию).</td>
  </tr>
  <tr>
    <td>$CVS_IGNORE_REMOTE_ROOT</td>
    <td>В соответствии с журналом изменений CVS, ChangeLog, данная переменная была удалена из CVS, начиная с версии 1.10.3.</td>
  </tr>
  <tr>
    <td>$CVS_PASSFILE</td>
    <td>Файл паролей для путей формата :pserver:. Эта переменная должна быть установлена до выполнения команды cvs login. По умолчанию это $HOME/.cvspass.</td>
  </tr>
  <tr>
    <td>$CVS_RCMD_PORT</td>
    <td>Порт для подключения к демону rcmd сервера для не-Unix систем.</td>

  <tr>
    <td>$CVS_RSH</td>
    <td>Программа для использования с путями формата :ext: в том случае, если это не rsh.</td>
  </tr>
