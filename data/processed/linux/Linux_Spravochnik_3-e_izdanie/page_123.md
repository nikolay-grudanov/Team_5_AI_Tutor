    <th>dip</th>
    <td><b>dip [options] [chat scriptfile]</b><br>
      Команда системного администрирования. Настройка и установка соединения с Интернетом. dip применяется для установки как исходящих, так и входящих соединений. Команды можно вводить в диалоговом режиме либо помещать в файл сценария для последующего выполнения при установке соединения. При обработке входящих соединений dip часто используется в качестве командного интерпретатора и может запускаться с помощью команд diplogin или diplogini.

      <b>Параметры</b>
      <ul>
        <li><b>-a</b> В режиме приема соединений запрашивать имя пользователя и пароль. Идентично команде diplogini.</li>
        <li><b>-i</b> Запустить интерпретатор для входящих соединений. Идентично команде diplogin.</li>
        <li><b>-k</b> Удалить самый последний процесс dip либо процесс, работающий на устройстве, которое задано параметром <b>-l</b>.</li>
        <li><b>-l device</b><br>
          Используется совместно с параметром <b>-k</b> для указания устройства терминала.
        </li>
        <li><b>-m mtu</b><br>
          Максимальный размер передаваемого блока (Maximum Transfer Unit). По умолчанию равен 296.
        </li>
        <li><b>-p protocol</b><br>
          Используемый протокол: SLIP, CSLIP, PPP или TERM.
        </li>
        <li><b>-t</b> Командный режим. Обычно применяется для проверки.</li>
        <li><b>-v</b> Режим диагностики.</li>
      </ul>

      <b>Команды</b>
      Большая часть этих команд могут выполняться как в диалоговом режиме, так и в файле сценария.

      <b>beep times</b><br>
      Выдать звуковой сигнал на терминал указанное количество раз.

      <b>bootp</b><br>
      Получить локальный и удаленный IP-адреса, используя протокол BOOTP.

      <b>break</b><br>
      Послать сигнал BREAK.
