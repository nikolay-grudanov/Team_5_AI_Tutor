    <th>xargs</th>
    <td>
      <b>-P max, --max-procs=max</b><br>
      Ограничить количество одновременно выполняемых заданий числом <i>max</i>. По умолчанию имеет значение, равное 1. Максимальное значение, равное 0, снимает ограничение на количество заданий.<br>
      <b>-r, --no-run-if-empty</b><br>
      Не выполнять команду, если она состоит только из пробелов.<br>
      <b>-s max, --max-chars=max</b><br>
      Ограничить длину командной строки количеством символов, равным <i>max</i>.<br>
      <b>-t, --verbose</b><br>
      Режим диагностики (отображать командную строку в стандартный поток ошибок перед выполнением).<br>
      <b>-x, --exit</b><br>
      Если превышен максимальный размер (заданный параметром <i>-s</i>), завершить работу.<br>
      <b>--version</b><br>
      Вывести номер версии xargs и завершить работу.<br>
      <b>Примеры</b><br>
      Производить поиск по шаблону <i>pattern</i> во всех файлах системы:
      <pre>find / -print | xargs grep pattern &gt; out &amp;</pre>
      Выполнить diff для пар файлов (например, f1.a и f1.b, f2.a и f2.b . . .):
      <pre>echo $* | xargs -n2 diff</pre>
      Предыдущая строка могла бы использоваться внутри сценария, вызываемого с именами файлов в качестве аргументов. В следующем примере указанный файл отображается целиком, по одному слову на строке (идентично работе команды deroff <i>-w</i>):
      <pre>cat file | xargs -n1</pre>
      Переместить файлы из каталога <i>olddir</i> в каталог <i>newdir</i>, показывая каждую команду переноса:
      <pre>ls olddir | xargs -i -t mv olddir/{} newdir/{}</pre>
