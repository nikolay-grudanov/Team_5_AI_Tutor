    <th>useradd</th>
    <td>
      <b>-m</b> Создать домашний каталог пользователя, если он еще не существует. По умолчанию домашний каталог не создается.<br>
      <b>-o</b> Принудительное использование указанного в параметре <i>-u</i> неуникального идентификатора пользователя (<i>uid</i>). (Не очень здравая мысль.)<br>
      <b>-s shell</b><br>
      Интерпретатор по умолчанию.<br>
      <b>-u uid</b><br>
      Численный идентификатор пользователя. Значение должно быть уникальным, если не указан параметр <b>-o</b>. Значение по умолчанию является самым маленьким значением, превосходящим 99 и любой другой идентификатор пользователя.<br>
      <b>-D [options]</b><br>
      Установить или отобразить значения по умолчанию. Если указаны параметры (<i>options</i>), устанавливать соответствующие значения, иначе отображать существующие.<br>
      <b>-b dir</b><br>
      Префикс, используемый при создании домашних каталогов. Если при создании записи не указан параметр <b>-d</b>, то к префиксу <i>dir</i> добавляется имя пользователя (<i>user</i>).<br>
      <b>-e date</b><br>
      Дата окончания доступа. Подразумевается применение зашифрованных паролей.<br>
      <b>-f days</b><br>
      Количество дней после устаревания пароля, на которое прекращается доступ пользователя к системе. Подразумевается, что пароли зашифрованы.<br>
      <b>-g group</b><br>
      Имя или идентификатор начальной группы.<br>
      <b>-s shell</b><br>
      Интерпретатор по умолчанию.
