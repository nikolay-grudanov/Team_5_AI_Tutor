Источник ввода и цель вывода могут быть переопределены при помощи команд перенаправления, перечисленных в следующих разделах.

Простое перенаправление

<table>
  <tr>
    <th>Команда</th>
    <th>Действие</th>
  </tr>
  <tr>
    <td>cmd &gt; file</td>
    <td>Перенаправить вывод команды cmd в указанный файл file (перезаписывается)</td>
  </tr>
  <tr>
    <td>cmd &gt;! file</td>
    <td>То же, но перезапись даже при установленном режиме noclobber</td>
  </tr>
  <tr>
    <td>cmd &gt;&gt; file</td>
    <td>Перенаправление вывода в файл file (добавление к существующему содержимому)</td>
  </tr>
  <tr>
    <td>cmd &gt;&gt;! file</td>
    <td>То же, даже при установленном режиме noclobber</td>
  </tr>
  <tr>
    <td>cmd &lt; file</td>
    <td>Перенаправление ввода команды из файла file</td>
  </tr>
  <tr>
    <td>cmd &lt;&lt; text</td>
    <td>Чтение со стандартного ввода вплоть до появления строки text (строка может быть значением переменной интерпретатора). Ввод обычно происходит на терминале или в процессе выполнения сценария. Как правило, такой синтаксис используют команды cat, echo, ex и sed. Если текст text заключен в кавычки, не происходит подстановки значений переменных для стандартного ввода, подстановок команд и т. д.</td>
  </tr>
</table>

Множественное перенаправление

<table>
  <tr>
    <th>Команда</th>
    <th>Действие</th>
  </tr>
  <tr>
    <td>cmd &gt;&amp; file</td>
    <td>Перенаправить стандартный вывод и поток ошибок в указанный файл</td>
  </tr>
  <tr>
    <td>cmd &gt;&amp;! file</td>
    <td>То же, даже при установленном режиме noclobber</td>
  </tr>
  <tr>
    <td>cmd &gt;&gt;&amp; file</td>
    <td>Добавление стандартного вывода и потока ошибок к содержимому указанного файла</td>
  </tr>
  <tr>
    <td>cmd &gt;&gt;&amp;! file</td>
    <td>То же, даже при установленном режиме noclobber</td>
  </tr>
  <tr>
    <td>cmd1 |&amp; cmd2</td>
    <td>Перенаправление стандартного вывода и потока ошибок через конвейер</td>
  </tr>
  <tr>
    <td>(cmd &gt; f1) &amp; f2</td>
    <td>Перенаправление стандартного вывода в файл f1, а стандартного потока ошибок в файл f2</td>
  </tr>
</table>
