    <th>dpkg</th>
    <td>
      <b>--largemem</b><br>
      Указание dpkg использовать столько памяти, сколько потребуется.<br>
      <b>--new</b><br>
      Новый формат пакетов с исполняемыми файлами. Это параметр dpkg-deb.<br>
      <b>--no-act</b><br>
      Выполнить все операции, но не записывать изменения. Используется для тестирования. Не забудьте указать этот параметр перед действием, в противном случае изменения могут быть записаны.<br>
      <b>--nocheck</b><br>
      Игнорировать содержимое управляющего файла при создании пакета. Это параметр dpkg-deb.<br>
      <b>-O, --selected-only</b><br>
      Обрабатывать только пакеты, которые отмечены для установки.<br>
      <b>--old</b><br>
      Старый формат пакетов с исполняемыми файлами. Это параметр dpkg-deb.<br>
      <b>-R, --recursive</b><br>
      Рекурсивная обработка файлов .deb, найденных в каталогах, указанных при помощи параметров -A, --install, --unpack или --avail, а также в подкаталогах.<br>
      <b>-R, --root=dir, --admindir=dir, --instdir=dir</b><br>
      Изменение каталогов по умолчанию. Каталог admindir содержит административные файлы с информацией о состоянии пакетов и т. д.; по умолчанию это /var/lib/dpkg. Каталог instdir — каталог, в который устанавливаются пакеты, по умолчанию — /. Изменение корневого каталога на dir автоматически изменит instdir на dir, а admindir — на /dir/var/lib/dpkg.<br>
      <b>--smallmem</b><br>
      Предписание dpkg экономить память при работе.
