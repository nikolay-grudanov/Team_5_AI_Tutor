    <th>export</th>
    <td>
      <b>export [options] [variables]</b><br>
      <b>export [options] [name=[value]] ...</b><br><br>
      Передать (экспортировать) значение одной или нескольких переменных окружения (<i>variables</i>), сделав переменные глобальными (по умолчанию они локальны). Например, в сценарии определяется переменная, которая должна быть экспортирована, чтобы другие программы, выполняемые сценарием, могли использовать ее. Если не заданы переменные, команда export перечисляет переменные, уже экспортированные в данном интерпретаторе. Если задано имя (<i>name</i>) и значение (<i>value</i>), происходит присваивание значения переменной с указанным именем.<br><br>
      <b>Параметры</b><br>
      —— Считать все последующие строки аргументами, а не параметрами.<br>
      -f Считать, что имена в списке <i>variables</i> являются именами функций.<br>
      -n Удалить переменную из области экспортированных.<br>
      -p Перечислить переменные, экспортированные в данном интерпретаторе.
    </td>
  </tr>
  <tr>
    <th>fc</th>
    <td>
      <b>fc [options][first][last]</b><br>
      <b>fc -e - [old=new][command]</b><br><br>
      Отобразить или редактировать команды из журнала команд (параметры —l и —e являются взаимоисключающими). Команда fc предоставляет возможности, аналогичные работе с журналом команд в интерпретаторе C shell и с командой !.<br>
      Параметры <i>first</i> и <i>last</i> являются номерами или строками, определяющими интервал отображения или редактирования команд. Если опущен параметр <i>last</i>, fc работает только с одной строкой (номер которой определяется параметром <i>first</i><sup>1</sup>). Если параметр <i>first</i> также опущен, происходит редактирование последней команды или перечисление последних 16 команд. Вторая форма использования fc позволяет восстановить команду из журнала, заменить строку <i>old</i> на строку <i>new</i>, а затем выполнить измененную команду. Если строки не заданы, команда просто выполняется повторно.
    </td>
  </tr>

<sup>1</sup> Но только при редактировании. Если указан параметр —l, опущенный параметр <i>last</i> приравнивается текущей команде. — Примеч. науч. ред.
