    <th>fuser</th>
    <td><b>fuser [options] [files | filesystems]</b><br><br>Идентификация процессов, использующих перечисленные файлы или файловые системы. Отображаемая информация интерпретируется следующим образом: <b>c</b> — текущий каталог, <b>e</b> — исполняемый файл, <b>f</b> — открытый файл, <b>m</b> — разделяемая библиотека, <b>r</b> — корневой каталог. Выполнять команду fuser разрешено пользователям, у которых есть права на чтение файлов /dev/kmem и /dev/mem, но принудительно завершать процессы, принадлежащие другим пользователям, может только привилегированный пользователь. fuser не работает с файлами на удаленных файловых системах (NFS).

Если указано более одной группы файлов, параметры могут быть заданы отдельно для каждой группы. Стоящий отдельно символ «—» (дефис) отменяет действие текущих параметров, а для следующей группы файлов вступают в силу новые параметры.

<b>Параметры</b><br>
— Вернуться к параметрам по умолчанию.<br>
—signal<br>
Послать процессу сигнал signal вместо SIGKILL.<br>
—a Отобразить информацию для всех перечисленных файлов, а не только для тех, с которым работает по меньшей мере один процесс.<br>
—i Запрашивать подтверждение пользователя для принудительного завершения процесса. Параметр игнорируется, если не указан также ключ —k.<br>
—k Послать сигнал SIGKILL всем процессам.<br>
—l Отобразить список имен существующих сигналов.<br>
—m Считать, что файлы (files) существуют на подмонтированной файловой системе; включать все файлы, принадлежащие этой файловой системе.<br>
—s Подавление вывода при поиске.<br>
—u Отображать имена владельцев процессов в скобках после идентификатора процесса.<br>
—v Подробная информация.<br>
—V Отобразить информацию о версии fuser.</td>
