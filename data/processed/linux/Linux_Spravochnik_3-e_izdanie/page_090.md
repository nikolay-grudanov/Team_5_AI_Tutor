<table>
  <tr>
    <th>—r, —R, —recursive</th>
    <td>Рекурсивное копирование каталогов.</td>
  </tr>
  <tr>
    <th>—S backup-suffix, —suffix=backup-suffix</th>
    <td>Установить символ суффикса, добавляемого при создании резервных копий к именам файлов. Суффикс также можно задать с помощью переменной окружения SIMPLE_BACKUP_SUFFIX. Суффикс по умолчанию — «~» (тильда). Для того чтобы отделить суффикс от имени, необходимо явно задать точку (т. е. .bak, а не bak).</td>
  </tr>
  <tr>
    <th>—s, —symbolic-link</th>
    <td>Вместо копий создавать символические ссылки для всех файлов. Для копируемых файлов должен быть задан полный путь.</td>
  </tr>
  <tr>
    <th>—u, —update</th>
    <td>Не копировать файл, если конечный файл уже существует и имеет такое же (или более позднее) время изменения.</td>
  </tr>
  <tr>
    <th>—v, —verbose</th>
    <td>Выводить имя каждого файла перед копированием.</td>
  </tr>
  <tr>
    <th>—V type, —version-control=type</th>
    <td>Определить тип создаваемых резервных копий. Можно также воспользоваться переменной окружения VERSION_CONTROL. По умолчанию имеет значение existing. Возможные аргументы:
      <ul>
        <li>t, numbered<br>Всегда создавать пронумерованные резервные копии.</li>
        <li>nil, existing<br>Создавать нумерованные резервные копии только для файлов, у которых они уже есть, в противном случае создавать обычные резервные копии.</li>
        <li>never, simple<br>Всегда создавать простые резервные копии.</li>
      </ul>
    </td>
  </tr>
  <tr>
    <th>—x, —one-file-system</th>
    <td>Игнорировать подкаталоги, расположенные в других файловых системах.</td>
  </tr>
  <tr>
    <th>cpio flags [options]</th>
    <td>Копирование файловых архивов с ленты (или диска) на ленту (или диск) либо в другое место на локальной машине. Каждый из трех флагов (flags) —i, —p и —o имеет свои параметры.</td>
  </tr>
