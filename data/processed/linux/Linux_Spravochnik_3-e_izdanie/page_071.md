    <th>bootptest</th>
    <td><b>bootptest [options] server [template]</b><br>
      Команда TCP/IP. Проверка работы демона <b>bootpd</b> сервера <i>server</i> путем посылки запросов каждую секунду в течение 10 секунд либо до отклика сервера. Если указан файл <i>template</i>, производится чтение настроек из этого файла.<br>
      <b>Параметры</b><br>
      <b>-f file</b><br>
      Прочитать имя файла загрузки из указанного файла.<br>
      <b>-h</b> Опознавать клиента по аппаратному, а не по IP-адресу.<br>
      <b>-m magic-number</b><br>
      Сделать число <i>magic-number</i> первым полем записи информации о производителе.

    <th>bzip2</th>
    <td><b>bzip2 [options] filenames</b><br>
      <b>bunzip2 [options] filenames</b><br>
      <b>bzcat [option] filenames</b><br>
      <b>bzip2recover filenames</b><br>
      Утилита сжатия и распаковки файлов, аналогичная gzip, но в ней для получения лучшего сжатия реализованы более совершенные алгоритмы и методы кодирования. <b>bzip2</b> заменяет каждый из перечисленных файлов сжатой версией, добавляя расширение .bz2. <b>bunzip2</b> производит распаковку файлов, сжатых <b>bzip2</b> (все прочие файлы игнорируются). <b>bzcat</b> распаковывает файлы на стандартный вывод, а <b>bzip2recover</b> используется для попытки восстановления данных из поврежденных упакованных файлов.<br>
      <b>Параметры</b><br>
      <b>--</b> Все последующие параметры считаются именами файлов.<br>
      <b>-dig</b><br>
      Размер блока при сжатии — <i>dig</i> × 100 Кбайт, где <i>dig</i> — цифра в интервале от 1 до 9.<br>
      <b>-c, --stdout</b><br>
      Сжатие или распаковка с записью полученных данных на стандартный вывод.<br>
      <b>-d, --decompress</b><br>
      Принудительная распаковка.
    </td>
