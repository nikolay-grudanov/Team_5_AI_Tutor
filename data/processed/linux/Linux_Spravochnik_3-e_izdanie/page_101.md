    <th>crontab</th>
    <td><b>crontab [options] [file]</b><br>
      Просмотреть, создать или удалить текущий файл конфигурации <i>crontab</i>. Привилегированный пользователь может запустить crontab для любого пользователя, указав параметр <b>-u user</b>. Файл конфигурации <i>crontab</i> представляет собой список команд (по одной на каждую строку), автоматически выполняющихся в определенное время. Время выполнения команд задается цифрами непосредственно перед каждой командой. Цифры, обозначающие время запуска команды, разделены на пять полей:

      <table>
        <tr>
          <th>Минута</th>
          <td>0–59</td>
        </tr>
        <tr>
          <th>Час</th>
          <td>0–23</td>
        </tr>
        <tr>
          <th>День месяца</th>
          <td>1–31</td>
        </tr>
        <tr>
          <th>Месяц</th>
          <td>1–12<br>Jan, Feb, Mar, ...</td>
        </tr>
        <tr>
          <th>День недели</th>
          <td>0–6, где 0 означает воскресенье<br>Sun, Mon, Tue, ...</td>
        </tr>
      </table>

      Несколько значений могут перечисляться через запятую, интервал обозначается с помощью дефиса, все возможные значения — символом звездочки (*). Предположим, файл <i>crontab</i> содержит следующие строки:

      <pre>
      59 3 * * 5   find / -print | backup_program
      0 0 1,15 * * echo "Timesheets due" | mail user
      </pre>

      Первая команда выполняет резервное копирование файлов системы каждую пятницу в 3:59 утра, а вторая отправляет уведомление почтой 1-го и 15-го числа каждого месяца.

      Привилегированный пользователь может выполнять команду crontab всегда. Прочие пользователи должны быть упомянуты в файле /etc/cron.allow, если таковой существует, в противном случае — не должны быть упомянуты в файле /etc/cron.deny. Если не существует ни один из файлов, команду может запускать только привилегированный пользователь.

      <b>Параметры</b>
      Параметры <b>-e, -l и -r</b> недопустимы, если задан хотя бы один файл в командной строке.
      <ul>
        <li><b>-e</b> Редактировать существующий файл <i>crontab</i> текущего пользователя (или создать его, если файл не существует).</li>
        <li><b>-l</b> Выдать файл <i>crontab</i> текущего пользователя на стандартный вывод.</li>
      </ul>
    </td>
  </tr>
