    <th>ypbind</th>
    <td><b>ypbind [options]</b><br>
      Команда NFS/NIS. Связывающий процесс NIS. ypbind — это демон, который обычно запускается при загрузке системы. Его задачей является запоминание информации, которая позволяет процессам клиентов на одном из узлов обмениваться данными с процессом ypserv. Данные, хранимые ypbind, называются связками (bindings). Они ассоциируют имя домена с интернет-адресом NIS-сервера и номером порта, на котором процесс ypserv этого узла осуществляет прием запросов клиентов. Данные хранятся в файле /var/yp/bindings/domainname.version.<br>
      <b>Параметры</b><br>
      <b>-ypset</b><br>
      Может использоваться для изменения связи. Применение параметра очень опасно и допустимо только при отладке работы сети с удаленной машины.<br>
      <b>-ypsetme</b><br>
      Запросы ypset могут выдаваться только с этой машины. Защита обеспечивается проверкой IP-адресов, что совершенно не помогает в случае посылки пакетов не заслуживающими доверия личностями. Не рекомендуется использовать этот параметр.
    </td>

    <th>ypcat</th>
    <td><b>ypcat [options] mname</b><br>
      Команда NFS/NIS. Вывод значения из базы данных NIS по имени mname (имя или псевдоним карты).<br>
      <b>Параметры</b><br>
      <b>-d domain</b><br>
      Указать домен вместо домена по умолчанию.<br>
      <b>-k</b> Отображать ключи для карт, в которых ключи имеют пустые значения или не являются частью значения.<br>
      <b>-t</b> Не переводить mname в имя карты.<br>
      <b>-x</b> Отображать таблицу псевдонимов, в которой перечисляются известные псевдонимы (mnames) и имена связанных карт. Аргумент mname при этом не требуется.
