    <th>strace</th>
    <td>отображается его имя, аргументы и возвращаемое значение, а также все сообщения об ошибках. Выводится имя сигнала и строка описания.<br><br>
<b>Параметры</b><br>
<b>-a n</b><br>
Выравнивать возвращаемые значения в колонке <i>n</i>.<br>
<b>-c</b><br>
Производить подсчет всех вызовов и сигналов и выдать сводку по завершении работы программы.<br>
<b>-d</b><br>
Режим отладки. Отладочная информация для strace выдается в стандартный поток ошибок (stderr).<br>
<b>-e keyword[=[!]values</b><br>
Передать strace выражение, определяющее, какие типы вызовов и сигналов отслеживаются и как. Значения выражений передаются в виде списка, элементы которого разделяются запятыми. Восклицательный знак (!) перед списком является отрицанием. Доступны ключевые слова all и none, а также нижеперечисленные:<br>
<b>abbrev=names</b><br>
Сокращать вывод крупных структур для системных вызовов, перечисленных в <i>names</i>.<br>
<b>read=descriptors</b><br>
Отображать все данные, которые прочитаны из перечисленных файловых дескрипторов.<br>
<b>signal=symbols</b><br>
Отслеживать сигналы с перечисленными именами (к примеру, signal=SIGIO,SIGHUP).<br>
<b>trace=values</b><br>
Отслеживать перечисленные значения. Значения могут включать имена системных вызовов либо один из следующих наборов системных вызовов:<br>
file   Вызовы с именем файла в качестве аргумента<br>
ipc    Взаимодействие процессов<br>
network Сетевые<br>
process Управление процессами<br>
signal Имеющие отношение к сигналам<br>
<b>verbose=names</b><br>
Для указанных системных вызовов отображать структуры полностью. По умолчанию имеет значение none.</td>
