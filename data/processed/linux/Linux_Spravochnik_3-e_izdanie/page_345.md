    <td>
      <b>-c name [login@]host[:dest]</b><br>
      Микровариант файла distfile. Аргумент login — имя пользователя для регистрации, host — целевой узел, name — имя передаваемого локального файла, а dest — конечное имя передаваемого файла в целевой системе.<br>
      <b>-d var=value</b><br>
      Определить переменную var со значением value. Параметр имеет больший приоритет, чем определения переменных в файле distfile.<br>
      <b>-f file</b><br>
      Считать данные из указанного файла (по умолчанию — distfile). Если имя указано символом «-» (дефис), выполняется чтение со стандартного ввода.<br>
      <b>-l options</b><br>
      Указать параметры регистрации на локальной машине.<br>
      <b>-m machine</b><br>
      Обновлять только для указанной машины machine. Параметр может указываться в нескольких экземплярах для различных машин.<br>
      <b>-n</b> Подавить нормальное выполнение. Отобразить команды, которые были бы выполнены.<br>
      <b>-o options</b><br>
      Задать один или более режимов, разделяемых запятыми.<br>
      chknfs<br>
      Не работать с файлами, которые находятся на файловых системах NFS.<br>
      chkreadonly<br>
      Перед обновлением файлов проверить, доступна ли система для записи.<br>
      chksym<br>
      Не обновлять файлы, которые существуют на локальном узле, но являются символическими ссылками на удаленном узле.<br>
      compare<br>
      Сравнивать файлы и использовать сравнения в качестве критериев принятия решения при обновлении (вместо возраста файлов).
