    <th>fetchmail</th>
    <td>
      <b>-e n, --expunge n</b><br>
      Послать IMAP-серверу команду EXPUNGE, стирающую отмеченные для удаления сообщения, после n удалений. Значение 0 означает стирание только в конце сеанса. Как правило, expunge выполняется при каждом удалении.<br><br>
      <b>-F, --flush</b><br>
      Для протоколов POP3 и IMAP: удалить уже полученные сообщения перед получением новых.<br><br>
      <b>-f file, --fetchmailrc file</b><br>
      Задать альтернативное имя файла настройки fetchmail.<br><br>
      <b>-I specification, --interface specification</b><br>
      Проверять, что почтовый сервер запущен и работает на указанном IP-адресе (или в диапазоне IP-адресов) перед его опросом. Спецификация (specification) указывается в виде interface/ipaddress/mask. Первая часть означает тип ожидаемого TCP-соединения (sl0, ppp0 и т. д.), вторая является IP-адресом, а третья — битовой маской IP, предположительно 255.255.255.255.<br><br>
      <b>-K, --nokeep</b><br>
      Удалять все полученные сообщения с почтового сервера.<br><br>
      <b>-k, --keep</b><br>
      Сохранять копии всех полученных сообщений на сервере.<br><br>
      <b>-l size, --limit size</b><br>
      Ограничить максимальный размер для получаемых сообщений. Сообщения, превышающие ограничение, будут оставлены на сервере и отмечены как непрочтенные.<br><br>
      <b>-M interface, --monitor interface</b><br>
      В режиме демона проверять указанный TCP/IP-интерфейс на наличие какой-либо внешней активности и пропускать опрос, если таковая отсутствует. Полезно для PPP-соединений, которые производят автоматическое рассоединение при отсутствии активности.<br><br>
      <b>-m command, --mda command</b><br>
      Передать почту напрямую агенту доставки почтовых сообщений, вместо того чтобы посылать их через 25 порт. Команда command является путем с параметрами вызываемого агента, например, /usr/lib/sendmail -oem. Параметр %T команды заменяется локальным адресом доставки, а %F — адресом отправителя сообщения.
