    <th>named</th>
    <td>В разных дистрибутивах Linux существует несколько исполняемых файлов named, поведение которых различается. Если ваша версия не работает так, как описано здесь, не тревожьтесь — к ней наверняка имеется документация.

<b>Параметры</b>

<b>-d debuglevel</b>
Отображение отладочной информации. debuglevel — это число, отражающее уровень подробности.

<b>-p port</b>
Использовать указанный порт. По умолчанию — 42.

<b>[–b] bootfile</b>
Файл, используемый вместо named.boot. Параметр –b является необязательным и предназначен только для задания имен, начинающихся с дефиса.

<b>Файл</b>
/etc/named.boot
Файл читается при запуске named.</td>
  </tr>
  <tr>
    <th>namei</th>
    <td><b>namei [options] pathname [pathname...]</b>
Следовать по пути, пока не найдена конечная точка (файл, каталог, файл символного устройства и т. д.). Найдя символьскую ссылку, namei отображает ее и следует по ссылке, отступами указывая контекст поиска. Если превышено максимально допустимое количество символьских ссылок в системе, namei выводит соответствующее сообщение.

<b>Параметры</b>

<b>-m</b> Отображать права доступа для файлов в стиле ls; например «rwxr-xr-x».

<b>-x</b> Отличать точки монтирования символом D вместо d.

<b>Символы типов файлов</b>
В каждую из строк вывода namei вставляет следующие символы для обозначения типа файла:
— Обычный файл.
? Ошибка какого-либо типа.
b Блочное устройство.</td>
