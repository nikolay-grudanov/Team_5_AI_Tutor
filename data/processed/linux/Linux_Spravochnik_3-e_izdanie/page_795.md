    <th>ci</th>
    <td>
      <b>-u[R]</b><br>
      Выполнить со <b>-u</b> после добавления файла. Это позволяет сохранить копию файла только для чтения.<br>
      <b>-wuser</b><br>
      Установить значение поля автора в user для добавляемой версии.<br>
      <b>Примеры</b><br>
      Добавить файлы глав, используя одинаковые записи в журнале:<br>
      <code>ci -m'First round edits' chap*</code><br>
      Добавить измененный файл prog.c, сохранив копию только для чтения:<br>
      <code>ci -u prog.c</code><br>
      Начать уровень издания 2; создать синоним «Prototype» для версии 2.1:<br>
      <code>ci -r2 -nPrototype prog.c</code>
    </td>
  </tr>
  <tr>
    <th>co</th>
    <td>
      <b>co [options] files</b><br>
      Извлечь одну из предварительно добавленных версий файла в соответствующий рабочий файл (либо вывести на стандартный вывод, если указан параметр <b>-p</b>). Если вы намереваетесь отредактировать рабочий файл и вновь добавить его, необходимо указать параметр <b>-l</b> для блокировки файла. co работает со стандартными параметрами <b>-q</b>, <b>-V</b> и <b>-x</b>.<br>
      <b>Параметры</b><br>
      <b>-d date</b><br>
      Извлечь самую позднюю версию, время добавления которой соответствует указанной дате <i>date</i> или ранее.<br>
      <b>-f[R]</b><br>
      Принудительная перезапись рабочего файла.<br>
      <b>-I[R]</b><br>
      Диалоговый режим; запрашивать ввод пользователя, даже если стандартным вводом не является терминал. (Параметр появился в RCS версии 5.)
    </td>
