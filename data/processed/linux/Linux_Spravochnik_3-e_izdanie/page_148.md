<table>
  <tr>
    <th>–follow</th>
    <td>Следовать по символическим ссылкам, отмечая каталоги как обработанные (не применять совместно с –type l).</td>
  </tr>
  <tr>
    <th>–group gname</th>
    <td>Найти файлы, принадлежащие группе gname. gname может быть как именем, так и идентификатором группы.</td>
  </tr>
  <tr>
    <th>–inum n</th>
    <td>Найти файлы, для которых число единиц inode равно n.</td>
  </tr>
  <tr>
    <th>–links n</th>
    <td>Найти файлы, на которые существует n ссылок.</td>
  </tr>
  <tr>
    <th>–mount, –xdev</th>
    <td>Искать только файлы, находящиеся внутри файловой системы, в которой начат поиск.</td>
  </tr>
  <tr>
    <th>–mtime +n | –n | n</th>
    <td>Найти файлы, которые в последний раз были модифицированы более чем n (+n) дней назад либо менее чем n (–n) дней назад, либо ровно n дней назад. Под модификацией понимается изменение содержимого файла.</td>
  </tr>
  <tr>
    <th>–name pattern</th>
    <td>Найти файлы, имена которых соответствуют шаблону (pattern). При использовании специальных символов необходимо экранировать их кавычками или символом «\».</td>
  </tr>
  <tr>
    <th>–newer file</th>
    <td>Найти файлы, содержимое которых изменялось позже, чем содержимое указанного файла (file); критерий схож с –mtime. –follow влияет на этот критерий, только если указан в командной строке раньше него.</td>
  </tr>
  <tr>
    <th>–ok command { } \;</th>
    <td>Идентично –exes, но перед выполнением команды запрашивается подтверждение пользователя.</td>
  </tr>
  <tr>
    <th>–perm nnn</th>
    <td>Найти файлы, восьмеричное значение прав доступа которых (например, rwx) в точности совпадает с nnn (например, –rw–rw–r— соответствует значению 664). Укажите знак «минус», чтобы произвести поиск по шаблону для любых незаданных прав доступа (например, –perm –600 отметит все файлы с правами доступа –rw–*******, где * может принимать любое значение).</td>
  </tr>
</table>
