    <td></td>
    <td><b>-b program version</b><br>
      Послать RPC-сообщение указанной программе с версией <i>version</i>, используя UDP, и отобразить список ответивших узлов.<br><br>
      <b>-d program version</b><br>
      Удалить указанную версию программы из списка зарегистрированных. Операция может выполняться только пользователем, добавившим программу, или привилегированным пользователем.<br><br>
      <b>-n portnum</b><br>
      Число <i>portnum</i> задает номер порта для параметров <b>-t</b> и <b>-u</b> вместо порта, указанного демоном portmapper.<br><br>
      <b>-p [host]</b><br>
      Послать запрос демону portmapper указанного узла и получить список всех зарегистрированных программ RPC. Если узел (<i>host</i>) не задан, используется значение, возвращаемое вызовом <b>hostname</b>.<br><br>
      <b>-t host program [version]</b><br>
      Произвести RPC-вызов программы (<i>program</i>) на указанном узле (<i>host</i>), используя TCP, и сообщить о получении ответа либо об отсутствии такового.<br><br>
      <b>-u host program [version]</b><br>
      Произвести RPC-вызов программы (<i>program</i>) на указанном узле (<i>host</i>), используя UDP, и сообщить о получении ответа либо об отсутствии такового.
    </td>
  </tr>
</table>

Примеры

Для отображения всех сервисов RPC, зарегистрированных на локальной машине, введите:

$ rpcinfo -p

Для отображения всех сервисов RPC, зарегистрированных на машине с именем klaxon, введите:

$ rpcinfo -p klaxon

Для перечисления всех машин локальной сети, на которых работает сетевая информационная служба (NIS), используйте команду:

$ rpcinfo -b ypserv version | uniq

где <i>version</i> — текущая версия NIS, информацию о которой можно получить, указав параметр <b>-p</b>.
