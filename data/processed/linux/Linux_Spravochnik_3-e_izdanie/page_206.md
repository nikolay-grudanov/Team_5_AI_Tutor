<h2>Alфавитный перечень команд</h2>
<table>
  <tr>
    <th>–V, --version</th>
    <td>Вывести информацию о версии hostname, завершить работу.</td>
    <th>hostname</th>
  </tr>
  <tr>
    <th>–y, --yr, --nis</th>
    <td>Отобразить имя NIS-домена. Привилегированный пользователь может установить новое имя при помощи аргумента nameofhost.</td>
    <th></th>
  </tr>
  <tr>
    <th>hwclock [options]</th>
    <td>Команда системного администрирования. Чтение или установка аппаратных часов. hwclock хранит в файле /etc/adjtime информацию об изменениях, которая может быть использована для подводки часов, если они отстают или спешиат. hwclock является заменой программы clock. Однобуквенные параметры включены для сохранения совместимости с прежней командой.</td>
    <th>hwclock</th>
  </tr>
  <tr>
    <th>Параметры</th>
    <td>Можно указывать лишь один из перечисленных ниже параметров:</td>
    <th></th>
  </tr>
  <tr>
    <th>–a</th>
    <td>Подводка аппаратных часов на основе информации из /etc/adjtime и установка системных часов на новое время.</td>
    <th></th>
  </tr>
  <tr>
    <th>--adjust</th>
    <td>Подводка аппаратных часов на основе информации из /etc/adjtime.</td>
    <th></th>
  </tr>
  <tr>
    <th>--date date</th>
    <td>Имеет смысл, только если параметр указан совместно с параметром --set. Аргумент date является строкой, пригодной к использованию с командой date.</td>
    <th></th>
  </tr>
  <tr>
    <th>--debug</th>
    <td>Диагностика действий hwclock.</td>
    <th></th>
  </tr>
  <tr>
    <th>–r, --show</th>
    <td>Вывести текущее время по аппаратным часам.</td>
    <th></th>
  </tr>
  <tr>
    <th>–s, --hctosys</th>
    <td>Привести системное время в соответствие с временем аппаратных часов.</td>
    <th></th>
  </tr>
