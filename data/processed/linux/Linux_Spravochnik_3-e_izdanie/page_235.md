    <td>
      Если существуют расширенные правила, связанные с данным протоколом, они будут загружены автоматически; нет необходимости загружать их явно с помощью параметра <b>-m</b>.

      <b>-s [!] address[/mask][!][port], --source [!] address[/mask][!][port]</b>

      Отбор пакетов, приходящих с указанного адреса. Адрес может указываться в виде имени узла, имени сети либо в виде IP-адреса. Необязательный параметр <i>mask</i> определяет сетевую маску и может представляться в классическом формате (например, /255.255.255.0) или в современной, более краткой форме (например, /24).

      <b>-d [!] address[/mask][!][port], --destination [!] address[/mask][port]</b>

      Отбор пакетов с определенным адресом назначения (<i>address</i>). Синтаксис параметра идентичен синтаксису для параметра <b>-s</b>.

      <b>-j target, --jump target</b>

      Переход к специальной цели либо определенной пользователем цепи правил. Если для правила этот параметр не определен, соответствие пакета правилу лишь увеличивает счетчики правила, а пакет передается для проверки на соответствие следующему правилу.

      <b>-i [!] name[+], --in-interface name[+]</b>

      Отбор пакетов, получаемых через интерфейс <i>name[+]</i>. <i>name</i> — это сетевой интерфейс, используемый системой (например, eth0 или ppp0). Для создания маски применяется символ «+». Так, значению ppp+ соответствуют все имена интерфейсов, начинающиеся с ppp.

      <b>-o [!] name[+], --out-interface name[+]</b>

      Отбор пакетов, отправляемых через интерфейс <i>name[+]</i>. См. описание синтаксиса <i>name</i> для параметра <b>-i</b>.

      <b>[!] -f, [!]--fragment $PARAMETER</b>

      Правило отбирает все, кроме первого, фрагменты разбитого на части пакета.

      <b>Параметры</b>

      <b>-v, --verbose</b>

      Режим подробной диагностики.
