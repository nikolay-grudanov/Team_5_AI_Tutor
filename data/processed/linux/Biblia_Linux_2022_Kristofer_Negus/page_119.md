Таблица 3.5. Общие переменные окружения

<table>
  <tr>
    <th>Переменная</th>
    <th>Описание</th>
  </tr>
  <tr>
    <td>BASH</td>
    <td>Отображает путь к оболочке, которая занимается интерпретацией команд. Обычно это путь /bin/bash</td>
  </tr>
  <tr>
    <td>BASH_VERSION</td>
    <td>Отображает конкретную версию bash</td>
  </tr>
  <tr>
    <td>EUID</td>
    <td>Идентификатор текущего пользователя. Он назначается при запуске оболочки на основе записи пользователя в файле /etc/passwd</td>
  </tr>
  <tr>
    <td>FCEDIT</td>
    <td>Редактор по умолчанию, используемый командой fc для работы с командой history. Если эта переменная не задана, берется команда vi</td>
  </tr>
  <tr>
    <td>HISTFIL</td>
    <td>Имя файла для сохранения списка истории оболочки (по умолчанию хранится в $HOME/.bash_history)</td>
  </tr>
  <tr>
    <td>HISTFILESIZE</td>
    <td>Максимальное количество строк, сохраняемых в файле истории. После достижения лимита самые старые команды удаляются. Лимит по умолчанию — 1000</td>
  </tr>
  <tr>
    <td>HISTCMD</td>
    <td>Номер истории текущей команды</td>
  </tr>
  <tr>
    <td>HOME</td>
    <td>Текущая домашняя папка пользователя. Это текущий рабочий каталог пользователя при входе в систему или применении команды cd с любыми параметрами</td>
  </tr>
  <tr>
    <td>HOSTTYPE</td>
    <td>Строка, описывающая компьютерную архитектуру устройства, на котором запущена операционная система. Для персональных компьютеров, совместимых с Intel, это i386, i486, i586, i686, например i386-linux. Для 64-разрядных машин AMD это значение x86_64</td>
  </tr>
  <tr>
    <td>MAIL</td>
    <td>Отображает электронную почту текущего пользователя. Это файл пользователя в каталоге /var/spool/mail</td>
  </tr>
  <tr>
    <td>OLDPWD</td>
    <td>Отображает предыдущий рабочий каталог, используемый в оболочке</td>
  </tr>
  <tr>
    <td>OSTYPE</td>
    <td>Строка, определяющая операционную систему, в которой работает оболочка. Для Fedora Linux значение OSTYPE равно linux или linux-gnu в зависимости от типа оболочки. (Bash может работать и в других операционных системах)</td>
  </tr>
  <tr>
    <td>PATH</td>
    <td>Список каталогов, которые будет проверять система при обращении к той или иной команде. Когда пользователь вводит команду, система проверяет указанные папки последовательно. Пример папки: /bin:/usr/bin:/usr/local/bin:/usr/bin/X11:/usr/X11R6/bin:~/bin. Если команды нет в переменной PATH, необходимо ввести полный или относительный путь к ней. Для суперпользователя значение также включает /sbin, /usr/sbin и /usr/local/sbin</td>
  </tr>
  <tr>
    <td>PPID</td>
    <td>Идентификатор родительского процесса (PID) оболочки bash (например, окно терминала, содержащее оболочку)</td>
  </tr>
</table>
