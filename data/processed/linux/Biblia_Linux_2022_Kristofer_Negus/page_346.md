Еще один вариант размонтирования занятого устройства — это параметр -l. С помощью umount -l («ленивое» размонтирование) оно происходит, как только устройство освобождается. Чтобы размонтировать удаленную файловую систему NFS, которая теперь недоступна (например, сервер вышел из строя), можно использовать параметр umount -f и сделать это принудительно.

СОВЕТ
Чтобы определить, чем занимается устройство, которое нужно размонтировать, существует полезный инструмент — команда lsof. Введите ее с именем нужного раздела (например, lsof /mnt/test). Выходные данные покажут, какие команды удерживают файлы открытыми в этом разделе. Таким же образом можно использовать команду fuser-v /mnt/test.

Создание файловой системы с помощью mkf

Вы можете создать файловую систему любого поддерживаемого типа на выбранном диске или в разделе с помощью команды mkfs. Этот вариант наиболее полезен для создания файловых систем в разделах жесткого диска, но его можно использовать и чтобы создавать файловые системы на USB-накопителях или перезаписываемых DVD.

Перед созданием новой файловой системы убедитесь в следующем.

● Вы разбили диск на разделы так, как нужно (с помощью команды fdisk).
● Вы выбрали правильное имя устройства (чтобы не перезаписать свой жесткий диск по ошибке). Например, первый раздел на втором SCSI или USB флешнакопителе в вашей системе — это /dev/sdb1, а третий диск — /dev/sdc1.
● Вы размонтировали раздел, если ранее он был смонтирован.

Далее приведены два примера использования команды mkfs для создания файловой системы в двух разделах USB-накопителя, расположенных как первый и второй разделы на третьем SCSI-диске (/dev/sdc1 и /dev/sdc2). Первый создает файловую систему xfs, а второй — ext4:

# mkfs -t xfs /dev/sdc1
meta-data=/dev/sda3 isize=256
        = sectsz=512
        = crc=0
data    = bsize=4096
        = sunit=0
naming  =version 2
log     =internal log
        = bsize=4096
realtime =none
        = sectsz=512
        = extsz=4096
agcount=4, agsize=256825 blks
attr=2, projid32bit=1
blocks=1027300, imaxpct=25
swidth=0 blks
ascii-ci=0 ftype=0
blocks=2560, version=2
sunit=0 blks, lazy-count=1
blocks=0, rtextents=0
