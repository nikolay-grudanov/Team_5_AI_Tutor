### Развертывание Jupyter Server в контейнере

С помощью этого руководства вы научитесь разворачивать Jupyter Server в контейнере. На примере развертывания Jupyter Server вы познакомитесь с созданием контейнера через интерфейс сервиса Container Apps и дополнительными настройками контейнера. Смите обучающее видео о Jupyter Server.

Вы будете использовать следующие сервисы:

- Artifact Registry для хранения, совместного использования и управления Docker-образами и Helm-чартами.
- Container Apps — сервис для запуска контейнерных приложений в облаке. Не требует знания Kubernetes и создания виртуальных машин.
- Систему контроля версий GitVerse. В GitVerse находится готовый образ Jupyter Server.

Шаги:

1. Подготовьте среду.
2. Клонируйте репозиторий кода с GitVerse.
3. Соберите образ, присвойте тег и загрузите образ.
4. Создайте и запустите контейнер.
5. Проверьте работу Jupyter Server.

Перед началом работы

Зарегистрируйтесь в личном кабинете Cloud.ru.

Если вы уже зарегистрированы, войдите под своей учетной записью.

1. Подготовьте среду
Подготовьте среду, если не сделали этого ранее.

2. Клонируйте репозиторий кода с GitVerse
Чтобы использовать образ Jupyter Server, склонируйте репозиторий:

```bash
git clone https://gitverse.ru/cloudru/evo-containerapp-jupyter-server-sample
```

3. Соберите образ, присвойте тег и загрузите образ
1. Перейдите в локальную папку с репозиторием:

```bash
cd evo-containerapp-jupyter-server-sample
```

2. Соберите образ:

```bash
docker build --platform linux/amd64 -t jupyter-server -f dist/jupyter-server/Dockerfile
```

Для создания контейнера Docker-образ должен быть собран под платформу linux/amd64, поэтому в команде используется флаг platform со значением linux/amd64.

1. Присвойте образу тег:

```bash
docker tag jupyter-server <registry_name>.cr.cloud.ru/jupyter-server
```

2. Загрузите образ в реестр.
Используйте реестр, созданный на этапе подготовки среды.

```bash
docker push <registry_name>.cr.cloud.ru/jupyter-server
```

Где:
- <registry_name> — название реестра, которое вы указывали при его создании в Artifact Registry.
- jupyter-server — название будущего репозитория в Artifact Registry. Название репозитория соответствует имени Docker-образа.

3. В личном кабинете перейдите в раздел Реестры → Репозитории → Артефакты сервиса Artifact Registry и убедитесь, что образ загружен.

![Screenshot of the Artifact Registry showing the jupyter-server artifact](https://i.imgur.com/3Q5z5QG.png)

4. Создайте и запустите контейнер
1. Перейдите в сервис Container Apps через меню в левом верхнем углу экрана.

![Screenshot of the Container Apps menu in the Evolution dashboard](https://i.imgur.com/7Q5z5QG.png)

2. Выберите Container Services и нажмите Создать.

Container Apps

Запустите ваше контейнерное приложение
Container Apps позволяет запускать контейнеры в готовом окружении. Вам больше не нужно создавать и поддерживать виртуальную инфраструктуру для работы приложения — контейнер запускается по запросу в готовой масштабируемой среде.
Для работы с Container Apps подключите к вашему проекту сервис Artifact Registry — приватный репозиторий для хранения Docker-образов.
Подробнее о работе с контейнерами читайте в документации.
