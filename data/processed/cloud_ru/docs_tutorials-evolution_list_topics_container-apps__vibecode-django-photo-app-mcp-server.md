**Веб-кодинг с помощью Foundation Models и подключение МСР-сервера для деплоя приложения в Container Apps**

С помощью этого руководства вы научитесь:
• веб-кодить базовое приложение на Python (фреймворк Django) с использованием VS Code и Foundation Models;
• создавать фронтенд с помощью готовых компонентов Foundation Models в VS Code;
• подключать кастомный AI-агент для работы с МСР-сервером в VS Code;
• автоматизировать деплои приложения в Container Apps, используя промпты к МСР-серверу.

Вы будете использовать набор готовых промптов всех шагов создания и деплоя приложения.
На примере этих промптов вы сможете не только нуля создать работающее приложение и запустить его в Container Apps, но и полностью автоматизировать процесс обновления и публикации новой версии приложения.

Вы будете использовать следующие сервисы:
• Artifact Registry для хранения, совместного использования и управления Docker-образами и Helm-артефактами.
• Container Apps — сервис для запуска контейнерных приложений в облаке. Не требует знания Kubernetes.
• Object Storage — объектное S3-хранилище с бесплатным хранением файлов, объемом до 15 ГБ.
• Foundation Models — сервис для доступа к API популярных фундаментальных моделей машинного обучения с открытым исходным кодом.
• Systemctl — инструмент для запуска и остановки сервисов.
• VS Code — редактор кода, который поддерживает множество языков программирования, включая Python, Java, C++, JavaScript и многие другие.
• Roo Code или Kilo Code — плагин для анализа, написания, рефакторинга и отладки кода. Поддерживает различные API и локальные модули. Позволяет создавать собственные AI-ассистенты для определенных задач и ролей, переключать режимы и настраивать промпты.

Примечание
Если вы хотите не писать Django-приложение с нуля, обращаясь к AI-модели с помощью промптов, попробуйте развернуть уже готовое приложение из репозитория GitVerse, перейдите к практическому руководству по развертыванию django-приложения.

В этом проекте используются:
• Django 3.2.7
• Python 3.13
• База данных SQLite
Внимание
Пример использования SQLite предназначен исключительно для демонстрационных целей и быстрого запуска приложения. Для продукционной среды настоятельно рекомендуется использовать полноценную СУБД, например PostgreSQL, она обеспечит надежность, масштабируемость и поддержку конкурентного доступа.
