ВАРИАНТ A - АВТОМАТИЧЕСКАЯ (РЕКОМЕНДУЕТСЯ):
────────────────────────────────────────────

$ python quickstart.py

Скрипт сам:
✅ Проверит Python версию
✅ Создаст виртуальное окружение (venv)
✅ Установит все зависимости (Flask, PyTorch, Transformers, и т.д.)
✅ Запустит тесты
✅ Запустит приложение

Время: 5-10 минут (зависит от интернета и ПК)


ВАРИАНТ B - РУЧНАЯ УСТАНОВКА:
──────────────────────────────

Шаг 1 - Создайте виртуальное окружение:
$ python -m venv venv

Шаг 2 - Активируйте (Windows PowerShell):
$ venv\Scripts\Activate.ps1

или Windows CMD:
$ venv\Scripts\activate.bat

или Linux/Mac:
$ source venv/bin/activate

Шаг 3 - Установите зависимости:
$ pip install -r requirements.txt

━━━━━━━━━━━━━━━━━━━━━━━━━━━

Если нет, выполните:

❌ ПРОБЛЕМА: "Python не найден"
✅ РЕШЕНИЕ:
   1. Установите Python с python.org
   2. При установке ОБЯЗАТЕЛЬНО отметьте "Add Python to PATH"
   3. Перезагрузитесь
   4. Откройте новую консоль PowerShell/CMD


❌ ПРОБЛЕМА: "Модули не установились"
✅ РЕШЕНИЕ:
   1. Обновите pip: python -m pip install --upgrade pip
   2. Переустановите: pip install -r requirements.txt
   3. Если медленно: pip install --default-timeout=1000 -r requirements.txt


❌ ПРОБЛЕМА: "Ошибка подключения к localhost:5000"
✅ РЕШЕНИЕ:
   1. Убедитесь, что приложение запущено в консоли
   2. Проверьте, нет ли ошибок в консоли
   3. Попробуйте другой браузер
   4. Очистите кэш браузера (Ctrl+Shift+Delete)


❌ ПРОБЛЕМА: "Первый запрос очень долгий"
✅ РЕШЕНИЕ:
   Это нормально! Первый запрос медленнее потому что:
   - Загружаются модели ML (~100 MB)
   - Создаются векторные представления
   - Инициализируется LLM
   
   Последующие запросы будут в 10-100x быстрее!


❌ ПРОБЛЕМА: "Материалы не найдены в ответах"
✅ РЕШЕНИЕ:
   1. Проверьте размер файла (не менее 500 символов)
   2. Убедитесь, что файл в правильной папке
   3. Проверьте кодировку UTF-8
   4. Перезагрузите приложение
   5. Очистите кэш: удалите data/vector_store/
   6. Перезапустите
